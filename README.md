# SynFact: Synthetic Data Generation for Knowledge Injection Benchmarks

A synthetic data generation engine for evaluating knowledge injection in LLMs via SFT and RL, supporting both text-only (SynFact-L) and multimodal (SynFact-VL) benchmarks.

## Installation

```bash
# Requires Python 3.11+
uv sync
```

## Benchmarks

### SynFact-L (Text-Only)

Generates fictional entities with structured relations, natural language descriptions, and QA pairs for evaluating factual knowledge grounding.

- **Connected World Generation**: Instead of isolated facts, SynFact generates a coherent **World Graph** where entities (Kingdoms, Planets, Characters) are interconnected.
- **Graph-Based Reasoning**: QA pairs are generated by traversing the knowledge graph, ensuring real multi-hop reasoning (e.g., questions about a neighbor's ruler).
## Configuration & parameters

User can control the **difficulty**, **quantity**, and **complexity** of the generated data using the following parameters:

| Parameter | Default | Description | Impact |
| :--- | :--- | :--- | :--- |
| `num_entities` | 1000 | Number of entities to generate | Controls the **Size** of the world graph. |
| `qa_pairs` | 5 | Direct QA pairs per entity | Controls volume of **Memorization** training data. |
| `ood_qa_pairs` | 5 | OOD QA pairs per entity | Controls volume of **Generalization** test data. |
| `max_hops` | 3 | Max reasoning hops | Controls **Reasoning Difficulty** for OOD questions. |
| `min_relations` | 8 | Min facts per entity | Controls **Graph Density** (sparsity). |
| `max_relations` | 15 | Max facts per entity | Controls **Graph Density** (richness). |
| `corpus_length` | "3-6" | Sentence count range | Controls **Context Length** (reading difficulty). |

## Quick Start
**Data generation (Full Example):**
```bash
uv run python scripts/run_synthesis_l.py \
    --num-entities 1000 \
    --qa-pairs 5 \
    --ood-qa-pairs 5 \
    --min-relations 10 --max-relations 20 \
    --max-hops 4 \
    --corpus-length "10-15" \
    --save-raw \
    --push-to-hub \
    --repo-id username/synfact-l
```

**Splits:**
- `train`: Full Description + Question → Answer
- `id_test`: Question → Answer (same QA as train, no context)
- `ood_test`: Question → Answer (implicit, inverse, multi-hop QA)

### SynFact-VL (Multimodal)

Generates fictional artworks with unique artists/styles, synthesized images, and visual QA pairs for evaluating visual knowledge grounding.

**Data generation:**
```bash
# With mock images (for testing)
uv run python scripts/run_synthesis_vl.py \
    --num-artworks 100 \
    --image-provider mock \
    --save-raw

# With real image generation
uv run python scripts/run_synthesis_vl.py \
    --num-artworks 100 \
    --image-provider nano_banana \
    --image-api-url "https://your-api.com" \
    --image-api-key "your-key" \
    --push-to-hub \
    --repo-id username/synfact-vl
```

**Splits:**
- `train`: Image + Full Caption + Question → Answer
- `id_test`: Image + Question → Answer (no caption)
- `ood_test_a`: Image + Unseen Questions → Answer
- `ood_test_b`: Artwork Title → Expected Caption (for reconstruction)

**Supported Image Providers:**
- `mock`: Placeholder images for testing
- `nano_banana`: Remote Nano-banana API
- `flux`: Local Flux model
- `qwen`: Qwen-Image model


## Configuration

Create a `.env` file:

```bash
SYNFACT_BASE_URL=https://your-llm-api.com/v1
SYNFACT_API_KEY=your-api-key
SYNFACT_MODEL_NAME=gpt-4o
```

## Upload to HuggingFace
```bash
uv run python scripts/upload_dataset.py \
    output/run_20260115_111842/synthesized_entities.json \
    --repo-id "YOUR_USERNAME/synfact-l-demo" \
    --length "10-15" \
    --max-hops 4 \
    --min-rel 5 --max-rel 10
```
